Use Case: Reset password
1.  User complete reset password form
2.  System validates reset password form
3.  System send reset password link email
4.  User click the reset password link
5.  User enters new password
6.  System stores new password
7.  System login user
Extensions
2a.  Invalid input entered
1.  System notifies user of invalid input
2.  User can then fix invalid input
5a.  Password to short or invalid
1.  System notifies user of invalid password
2.  User can then enter a valid password
